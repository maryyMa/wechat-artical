# 文章框架库

框架不是套模板，而是决定读者的阅读路径。选框架时看选题本身的张力、证据和情绪走向。

## 快速选择

| 框架 | 适合什么选题 | 不适合什么 |
|------|--------------|------------|
| 痛点型 | 读者已经有明确困扰，需要解决方案 | 纯观点、纯新闻 |
| 故事型 | 有人物、场景、转折、经历 | 缺少真实细节 |
| 清单型 | 需要给方法、步骤、判断标准 | 洞察不足时容易水 |
| 对比型 | 两种选择、两类人、两个趋势之间有冲突 | 没有可比对象 |
| 热评型 | 热点刚发生，需要表达立场 | 没有独特判断 |
| 揭露型 | 常识背后有机制、误区或利益结构 | 证据不足 |

## 痛点型

结构：

1. 镜子式开头：让读者看到自己的困境。
2. 确认感：告诉读者“不是你一个人这样”。
3. 问题拆解：把痛点拆成 2 到 4 个层次。
4. 解决阶梯：给出判断标准或行动步骤。
5. 证据：案例、数据或经验。
6. 收束：一句有力结论。

金句适合放在问题拆解后，作为认知转折。

## 故事型

结构：

1. 闪回式开头：先给结果或冲突。
2. 场景铺垫：补充背景和 stakes。
3. 转折：事情不是表面看起来那样。
4. 洞察浮现：从故事中长出观点。
5. 迁移：把个体故事推到读者处境。
6. 结尾：留一个余味，而不是总结全文。

故事型最怕编造和煽情。没有细节时不要硬写故事。

## 清单型

结构：

1. 开头指出问题。
2. 给出清单的使用场景。
3. 每一点都包含：判断、例子、误区、行动。
4. 结尾告诉读者先做哪一步。

清单不要全部等长。重点项应该更厚，辅助项可以短。

## 对比型

结构：

1. 提出两种常见选择或两类人。
2. 表面对比。
3. 隐藏变量。
4. 为什么大多数人会选错。
5. 新判断标准。
6. 回到读者该怎么做。

适合标题里出现“不是 A，而是 B”的选题。

## 热评型

结构：

1. 用最短篇幅交代热点。
2. 亮出判断，不要绕。
3. 解释为什么这个判断和主流不同。
4. 给出证据和反例。
5. 承认一个反方可能正确的部分。
6. 最后把热点上升到更长期的问题。

热评型必须快，但不能浅。没有新判断就不要写。

## 揭露型

结构：

1. 指出一个大家习以为常的现象。
2. 说明它为什么不正常。
3. 拆背后的机制。
4. 展示谁受益、谁付出成本。
5. 给读者一个新的识别方式。
6. 收束到行动或认知。

揭露型需要证据。没有证据时要降级成观点型，不要装成调查。

## 框架输出格式

为每个候选框架输出：

- 框架名
- 推荐分
- 开头策略
- 情绪曲线
- H2 大纲
- 金句位置
- 结尾方式
- 风险提醒
