# 写作指南

这份指南用于 Step 4 写正文。目标不是“写得像 AI 很会总结”，而是写得像一个有判断、有经验、有语气的人。

## 写作前先找核心洞察

核心洞察是一句话：读者看完后，会愿意转述给朋友的那句话。

坏例子：

- AI 正在改变工作方式。
- 普通人要拥抱变化。
- 企业需要提升效率。

好例子：

- 很多团队不是不会用 AI，而是把 AI 塞进旧流程里，所以越自动化越混乱。
- 年轻人害怕的不是加班，而是努力没有进入任何可积累的系统。

如果一句话说不清核心洞察，不要开始写正文。

## 写作前思考框架

写正文前先完成这些判断：

1. 读者是谁，正在为什么事情烦。
2. 这篇文章要改变读者的哪个判断。
3. 读者现在相信什么，但那只对了一半。
4. 文章最强的例子是什么。
5. 哪一段应该成为截图金句。
6. 文章情绪曲线是什么。

## 深度结构

每个关键观点至少走到第三层。

```text
第一层：发生了什么
第二层：为什么会这样
第三层：这对读者意味着什么
第四层：读者接下来该怎么判断或行动
```

文章至少两处达到第三层，否则只是信息整理。

## 情绪曲线

公众号文章需要节奏。常见曲线：

- 痛点 -> 松一口气 -> 更深问题 -> 解决方案
- 好奇 -> 惊讶 -> 认同 -> 行动
- 愤怒 -> 冷静拆解 -> 重新判断
- 故事 -> 转折 -> 洞察 -> 余味

情绪峰值通常放在 60% 到 70% 处。太早会后劲不足，太晚读者可能已经离开。

## 去 AI 味协议

### 第一层：删空话

删除这些句式：

- 在当今时代
- 值得注意的是
- 不难发现
- 总的来说
- 这不仅是……更是……
- 让我们一起……

替换成具体场景、具体动作、具体判断。

### 第二层：打破工整

AI 喜欢三段式和完美排比。真人表达通常有轻重。

检查：

- 是否连续三段差不多长。
- 是否每段都用“首先、其次、最后”。
- 是否每个小标题都像报告。
- 是否每个例子承担同一个功能。

修法：

- 两个重点足够时不要硬凑三个。
- 一个段落可以只有一句。
- 重要例子写厚，辅助例子写短。
- 小标题要有悬念或判断。

### 第三层：加入认知痕迹

真人写作会留下思考过程：

- 我原来以为……
- 后来发现不对。
- 这件事最反直觉的地方是……
- 有个细节很容易被忽略。

不要写成“标准答案”，要写成“一个人想明白了什么”。

### 第四层：最终声音检查

写完后逐段检查：

- 这段能不能套到另一个话题？能的话重写。
- 有没有具体人、具体场景、具体动作？
- 有没有至少一个反常识判断？
- 有没有连续三段长度相近？
- 读每段第一句，像不像提纲？
- 能不能删掉 20% 形容词而不损失意思？

## 结构要求

| 项目 | 要求 |
|------|------|
| 总字数 | 1500 到 2500 字，1500 到 2000 字通常更稳 |
| H1 标题 | 20 到 28 个汉字 |
| H2 小标题 | 至少 3 个，要有判断或悬念 |
| 段落长度 | 单段尽量不超过 150 字 |
| 钩子间隔 | 每 3 到 4 段要有一个转折、例子或金句 |
| 视觉节奏 | 不要连续 7 到 10 行都是纯文字 |
| 截图段落 | 至少 1 段，最好 2 到 3 段 |

## 编辑命令

| 用户说 | 执行方式 |
|--------|----------|
| 润色 | 保留内容和结构，优化表达，削弱空话 |
| 缩写 | 压到 60% 到 70%，保留核心洞察和最强例子 |
| 扩写 | 增加例子、层次、反方视角，不加水 |
| 换语气 | 按用户指定语气重写，需要明确目标语气 |
| 重写 | 保留主题，重新设计结构和表达 |

## 常见失败模式

泛泛洞察：结论人人都知道。解决：继续追问“所以呢”。

平衡过度：每边都说得有道理，最后没有立场。解决：承认反方一部分正确，但给出自己的判断。

总结式结尾：把文章又复述一遍。解决：结尾给新东西，不重复。

伪散文清单：把 bullet 改成“首先其次最后”。解决：让观点互相推进，而不是并排摆放。

开头太慢：300 字都没进入主题。解决：从最有张力的地方开始。

情绪平线：每段强度一样。解决：安排紧张、释放、惊讶、反思。
