# 微信公众号技术限制

微信公众号编辑器和渲染环境会过滤很多 HTML/CSS。排版时必须按微信安全规则来。

## 不可靠或不支持的 CSS

- 外链样式和 `<style>` 块
- `position: fixed` / `sticky`
- `transform` / `animation` / `transition`
- `filter` / `backdrop-filter`
- `@font-face`
- CSS Grid
- 部分 Flexbox
- CSS Variables
- Media queries

## 会被过滤的 HTML

- `<script>`
- `onclick`、`onload` 等事件属性
- `<video>`、`<audio>`
- `<iframe>`
- `<form>`
- SVG 支持不稳定

## 图片限制

- 本地路径不能直接发布，必须上传成微信图片 URL。
- 单张图片建议小于 5MB。
- WebP 在部分设备上不稳定，优先 JPG/PNG。
- 封面建议 900×383，约 2.35:1。

## 转换器做了什么

工具里的转换器会：

1. 把样式转成内联 `style`。
2. 只使用微信相对安全的 CSS。
3. 处理代码块，使用 `white-space: pre-wrap`。
4. 发布时上传本地图片，替换成微信 URL。
5. 表格尽量控制在移动端可读范围。
6. 给每个段落设置显式颜色，避免微信继承异常。
7. 使用系统字体栈，不依赖自定义字体。

## 内容大小建议

| 项目 | 建议/限制 |
|------|-----------|
| 草稿标题 | 控制在微信限制内，中文标题 20 到 28 字较稳 |
| 摘要 | 不超过 120 UTF-8 bytes，约 54 个汉字 |
| 正文 HTML | 小于 2MB |
| 单张图片 | 小于 5MB |
| 图片数量 | 不超过 20 张 |
| 封面 | 900×383 推荐 |

## 排版参数

| 参数 | 推荐值 |
|------|--------|
| 正文字号 | 15 到 16px |
| 注释字号 | 12 到 14px |
| H2 字号 | 17 到 24px |
| 行高 | 1.6 到 2.0 |
| 段落间距 | 12 到 28px |
| 页面边距 | 8 到 20px |

## 颜色规则

- 正文不要用纯黑 `#000000`，推荐 `#2c2c2c` 到 `#444444`。
- 主色用于标题、强调、装饰。
- 辅助色用于注释、引用、说明。
- 高饱和颜色不要大面积用于正文。
- 链接可以用下划线，普通强调少用下划线。

## 留白规则

- 小标题上方留更多空间，下方少一点。
- 每 3 屏左右可以出现一张图或一个明显视觉分隔。
- 不要连续 7 到 10 行都是纯文字。
